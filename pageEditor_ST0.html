CV Page

Technical Description

For clarity and accessibility, this HTML CV page is organized using semantic HTML5 elements such as <header>,<main>and <footer>. It employs <section> ,<dl>, <dt> ,<dd> tags to arrange content under Education, Work-Experience, Skills and certifications, and it has a navigation bar and internal anchor links for seamless section jumps. Layout, background, and alignment are managed by the internal CSS styles, particularly when using Flexbox for heading sections.
The page is visually styled with consistent font, color, and spacing, and images are used as icons next to each section. The page successfully blends structure and design, making it readable and generally well-organized.

Accessibility

In order to assist users with disabilities, this page adheres to a number of important accessibility guidelines. Screen readers can better understand the page structure with the aid of semantic tags.
Every image has alt attributes, which guarantee that visually impaired users are able to understand important visuals. Keyboard users can navigate more easily with internal anchor links, which enable fast section jumps without the need for a mouse.
To improve usability, text links are styled with visible color and hover effects. The layout uses CSS for spacing instead of a lot of <br>tags, which is better for responsive design and screen reader interpretation.

Ideal Job Page

Technical Description

This HTML document uses a responsive navigation system along with semantic HTML5 elements <header>,<main> ,<section>,<footer>to create a well-structured web page that describes the user's ideal job. It applies table formatting using internal styles to visually arrange a skill mapping matrix and combines external and internal CSS to guarantee modular styling.
For a clean layout and better readability, important web development best practices are used, such as border-collapse, consistent padding, and color differentiation for headers. For logo branding, image elements are suitably embedded with alt attributes, and navigation is controlled by an unordered list. To preserve semantic clarity and enhance structural logic, the table makes use of <caption>, <th rowspan>, and <th colspan>.

Accessibility

A number of accessibility-conscious elements are included on the page to help screen reader users, including descriptive alt text for the logo image, semantic HTML elements for a clear structure, and meaningful section headings. An unordered list is used to organize the navigation menu, which enhances keyboard navigation.
To guarantee that screen readers correctly interpret the data, table headers use the proper rowspan and colspan attributes.
Readability is improved by the contrast between font and background colors (#0077cc for headers, #f9f9f9 for cells). Furthermore, links have clear labels, and the use of semantic tags enhances compatibility with assistive technology, guaranteeing an inclusive experience for users with disabilities.

Query Page

Technical Description

Semantic HTML5 is used to construct the query form page, and both internal and external CSS are used for styling. <fieldset> and <legend> elements are used in the form's structure to logically group related form controls, improving readability and organization. Text, email, phone, date pickers, dropdowns, and radio buttons are examples of input fields.
Each of these is properly labeled with the <label> element and has a corresponding id attribute. A responsive, user-friendly layout with precisely defined button spacing, padding, and hover states is made possible by internal CSS. HTML5 validation properties like required, type="email," and placeholder hints limit all user inputs, guaranteeing clean and legitimate data submission even prior to server-side processing.

Accessibility

The form uses semantic HTML and appropriate form labeling to prioritize accessibility. The for attribute is used to pair each input element with a <label>, which enhances screen reader compatibility and navigation. Form sections can be meaningfully interpreted by assistive technologies thanks to the sensible use of <fieldset> and <legend>. For users with visual impairments, the color scheme ensures readability by maintaining good contrast (e.g., dark blue on light gray).
Grouped selection is made possible by the radio buttons' use of shared name attributes and the keyboard accessibility of the input fields. For the benefit of both screen reader users and keyboard-only navigators, required fields are visually indicated with a red asterisk and programmatically enforced with the required attribute.

Sitemap Page

Technical Description

A responsive SVG-based layout is used on the sitemap page to graphically depict the website's organization. elements styled with CSS and wrapped in <a> tags for linking to specific pages are used to create interactive navigation blocks. These blocks are labeled with <text> elements for readability and are positioned precisely using x and y coordinates.
Hover transitions, stroke outlines, and fill colors are all made consistent by internal CSS styles. The viewBox attribute in the SVG and flexbox in the.sitemap-container work together to guarantee that the diagram stays centered and scalable on all screen sizes. To create a logical visual flow between related pages, connector lines () are added.

Accessibility

Utilizing SVG accessibility features, like <title> elements inside each <a> to provide screen reader-friendly tooltips, takes accessibility into account. Users can navigate with just their keyboards thanks to the keyboard-navigable nature of all interactive elements (the page blocks) via tabindex="0".
The use of semantic <header>, <main>, and <footer> tags enhances screen reader interpretation of the page structure, and color contrasts between text and backgrounds are adequate to assist users with visual impairments. To avoid unintentional interaction, pointer events are disabled on non-interactive text and readable fonts and sizes are used throughout the SVG, concentrating interactivity on the links themselves.

Challenges and Lessons Learned

Making the SVG sitemap keyboard accessible and responsive was one of the biggest challenges. I fixed this by using `tabindex` on each link and adding a `viewBox` to the SVG. Additionally, I improved my ability to distinguish between page-specific modifications (internal CSS) and layout styling (external CSS).

Compliance

Each page validates without HTML errors and conforms with semantic HTML5. They meet Janet's standards for accessibility, appropriate linking, consistent navigation, and sound structure.JavaScript was not used.
